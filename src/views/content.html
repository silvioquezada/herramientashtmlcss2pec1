Aprender Programación

Soy Silvio Quezada tengo una tecnología en análisis de sistemas y tanbién tengo una licenciatura en ciencias de la educación, esta tecnología me permitió aprender conceptos, fundamentos y procesos del desarrollo de software, en ese tiempo aprendí más en el desarrollo de aplicaciones de escritorios con manejo a bases de datos, aún no se conocían muchos las tecnologías del desarrollo web, eran más populares la de escritorio, en ese entónces aprendí a utilizar Visual Basic 6.0 y bases de datos en MySql, sabía algo básico de PHP y un poco de Visual Basic .Net, lo básico.

Poco a poco fui aprendiendo el desarrollo de software por mi cuenta, esta es mi afición querer aprender en mis tiempos libres programación, ayudándome de documentación en la web, y cursos, aprendí PHP código espagueti porque estaba aprendiendo por mi cuenta, no sabía que arquitectura utilizar y tecnologías a utilizar, así que aprendí a hacerlo con 100% php y me faltaba algo para que sea dinámico así que aprendí JavaScript pero terminé utilizando JQuery, me enfoqué mas a las aplicaciones de gestión de información como facturación, gestión académica, no dominada los estilos css, sabía lo básico pero desconocía totalmente de las buenas prácticas en Html y Css incluso en código JavaScript, luego descubrí la librería Bootstrap y aprendía autilizarla sin conocer el css detrás de esta librería, era fantástico hacer formularios bien definidos así que no aprendí bien css.

Quice dar un salto aprender programación movil en android studio pero los conceptos orientados a objetos no me permitía leer el código, desconocía totalmente la programación orientada a objetos, así qu no podía aprender directo desarrollo movil sin aprender a utilizar los conceptos en Java de POO. Aprendí Java orientado a objetos desde lo básico hasta las conexiones de bases de datos terminando haciendo un CRUD y una aplicación pequeña de ventas como práctica, esto me permitió comprender código y prácticar desarrollo móvil, así se me hizo fácil leer código orientado a objetos.

Como mi código estaba desordenado aprendí la arquitectura MVC, y poder organizar el código tanto en Java y aplicacicones en PHP, todo me manejaba sin framework. Con esto mismo aprendí Visual C# aplicando conceptos orientado a objetos y MVC, tanto escritorio y en web.

He desarrollado aplicaciones reales luego de eso como:

  • Sistema de gestión académica.
  • Sistema de facturación electrónica.
  • Sistema de control médico.

Actualmente el que más le he dado soporte es al sistema de facturación electrónica, las aplicaciones funcionan y han ido mejorando poco a poco.

Con la maestría me he dado en cuenta que no he seguido las buenas prácticas tanto en html y css, además no sabía trabajar correctamente, no sabía utilizar entornos de trabajos nuevos, si hubiese seguido primero la maestría antes de desarrollar aplicaciones reales mis aplicaciones fueran escalables mi código más limpio, bien organizado, programando correctamente aplicando las buenas prácticas.
